## 1. EXECUTIVE SUMMARY

- **Compound pages are world-class.** `/bpc-157` ships 7 schema types (MedicalWebPage + DietarySupplement + BreadcrumbList + FAQPage + MedicalStudy[] + Dataset + DefinedTerm) plus Organization, full canonical, real PubChem/CAS/MeSH `sameAs` wiring, citations with `datePublished` + `author`, evidence-grade Dataset. This is the single biggest moat the property has.
- **Homepage is the weak link.** Production HTML has NO `application/ld+json` script in `<head>`, NO `rel="canonical"` link, NO inline `<h1>` in the static markup (h1 renders post-hydration via RSC payload). The Org+WebSite JSON-LD literally exists in `src/app/layout.tsx` source but the deployed build is not emitting it. Could be Cloudflare cache stripping or a Next.js streaming issue. **This is the highest-ROI fix on the entire property.**
- **`wip-clean-seo-rebuild` queues massive wins.** `/peptides/[slug]` URL restructure + 81 permanent 301s + ImageObject + SpeakableSpecification + per-page OG image generation + IndexNow ping route + 13 new `/vs/` comparison pages + Wikidata draft + entity IDs filled for 17 compounds. **Merge this branch ASAP.**
- **One genuinely broken thing.** BPC-157 page sends `og:image` pointing to `/cards/retatrutide.png` and `<img alt="BPC-157 trading card">` while `src=/cards/retatrutide.png`. Card-to-slug mapping is wrong somewhere. This is a real bug, not just suboptimal.
- **Off-page is the next bottleneck.** Technical-on-page is near-saturated. Wikidata Q-items (P-5 pending), ORCID for editorial board, HARO seeding, Wayback archive, and AI-search 40+ mention threshold are now the limiting factor on AI recommendation eligibility.
